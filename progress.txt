============================================================
KEY DESIGN DECISIONS
============================================================

- Password: PBKDF2-SHA256, 100k iterations, 16-byte salt, 32-byte hash
- Session in SecureStorage (keys: abyss_user_id, abyss_username,
  abyss_active_player_id)
- BackupData = User + Players + Inventory + Journals + SaveData +
  StoryProgress, SchemaVersion=1.0 for future migration
- StoryEvent/StoryChoice are POCOs (JSON, NOT in DB)
- Item is master data, InventoryItem is per-player instance with
  ItemData [Ignore] populated by InventoryService.GetByPlayerAsync
- Journal types: Story (auto-unlock, read-only) and Player (CRUD)
- ChoiceOutcome reports: Success/Reject/HpDelta/GoldDelta/ExpDelta/
  ItemAcquired/ItemConsumed/LeveledUp/PlayerDied/GameCompleted/
  EndingType/ShouldAdvanceFloor/StoryJournalUnlocked
